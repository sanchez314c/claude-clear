## Documentation Standards

### File Naming Conventions
- **UPPERCASE.md**: Core project documentation (root level)
- **lowercase.md**: Standard documentation files
- **CamelCase.md**: Specialized documentation
- **README.md**: Always project overview
- **CHANGELOG.md**: Always version history

### Documentation Format
- **Markdown**: All documentation uses GitHub-flavored markdown
- **TOC**: Table of contents for long documents
- **Links**: Cross-references between related documents
- **Code Blocks**: Syntax-highlighted examples
- **Tables**: Structured information presentation

### Maintenance Guidelines
- **Updates**: Keep documentation in sync with code changes
- **Reviews**: Regular documentation reviews for accuracy
- **Archive**: Move outdated docs to archive directory
- **Index**: Update this index when adding/removing documents
